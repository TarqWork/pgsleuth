| **eBPF Program Type**          | **Attachment Point**         | **Associated Data Context**   | **Description**                                                                                                                                                     |
| ------------------------------ | ---------------------------- | ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `BPF_PROG_TYPE_KPROBE`         | Dynamic Kernel Functions     | `pt_regs` (CPU Registers)     | Intercepts almost any kernel function upon entry (`kprobe`) or exit (`kretprobe`). You must manually parse CPU registers to extract function arguments.             |
| `BPF_PROG_TYPE_TRACEPOINT`     | Static Kernel Tracepoints    | Defined Tracepoint Struct     | Hooks into stable, pre-defined markers in the kernel. The data is pre-formatted by the kernel into a predictable struct containing event-specific fields.           |
| `BPF_PROG_TYPE_RAW_TRACEPOINT` | Raw Static Tracepoints       | Raw memory arguments          | Similar to tracepoints but bypasses the kernel's formatting step, offering lower overhead at the cost of needing to parse raw memory structures manually.           |
| `BPF_PROG_TYPE_UPROBE`         | User-Space Functions         | `pt_regs` (CPU Registers)     | Attaches to functions in user-space application binaries or shared libraries. Like kprobes, data is extracted directly from CPU registers.                          |
| `USDT` (via Kprobe/Uprobe)     | User-Space Static Probes     | Application-defined arguments | User Statically-Defined Tracing relies on developers embedding explicit tracepoints into application binaries. Exposes custom application state.                    |
| `BPF_PROG_TYPE_LSM`            | Linux Security Modules       | Kernel object pointers        | Attaches to MAC (Mandatory Access Control) security hooks. Provides access to sensitive kernel objects (inodes, credentials, sockets) to enforce security policies. |
| `BPF_PROG_TYPE_PERF_EVENT`     | Hardware/Software Counters   | `bpf_perf_event_data`         | Triggered by hardware performance counters (e.g., cache misses, branch mispredictions) or software events (e.g., page faults).                                      |
| `BPF_PROG_TYPE_XDP`            | Network Interface Card (NIC) | `xdp_md` (Raw packet data)    | Executes at the driver level before the kernel allocates memory for a packet. Used for extreme high-performance firewalls and load balancers.                       |
| `BPF_PROG_TYPE_SCHED_CLS`      | Traffic Control (TC)         | `__sk_buff` (Socket buffer)   | Operates on the kernel's ingress/egress network scheduling layer after packet memory is allocated.                                                                  |
| `BPF_PROG_TYPE_SOCKET_FILTER`  | Raw Sockets                  | `__sk_buff` (Socket buffer)   | Filters data packets directly at the network socket layer.                                                                                                          |
| `BPF_PROG_TYPE_CGROUP_SKB`     | cGroup Network Traffic       | `__sk_buff` (Socket buffer)   | Intercepts ingress or egress traffic exclusively for processes residing within a specific Linux Control Group.                                                      |
| `BPF_PROG_TYPE_CGROUP_SOCKOPT` | cGroup Socket Options        | `bpf_sockopt`                 | Allows eBPF to inspect or modify `setsockopt` and `getsockopt` calls for processes in a cGroup.                                                                     |
